Honeypot project

This project was a pivotal component of my Web Technology, Security, and Honeypot course, guided by Mr. Koreman, who provided comprehensive insights into web security, effective monitoring with an ELK stack, and the streamlined setup of environments using Ansible.

In groups of three, our task was to develop a website featuring a Capture The Flag (CTF) challenge. The website hosted five challenges, and participants were encouraged to attempt to solve them, even with the permission to deliberately break the application. As developers, our responsibility extended to ensuring the website's security, actively preventing any attempts at hacking.

The website, designed with a standard layout, presented five doors, each representing a unique challenge. The challenges included exploiting a PHP vulnerability to log in as an admin without knowing the password, leveraging this vulnerability to achieve Server-Side Request Forgery (SSRF), uploading an image to trigger Cross-Site Scripting (XXS), bypassing the "not allowed" page, and discovering a hidden page. Mats Fiers took charge of programming the PHP part, while I established the database on the server.

Following the website's creation, I developed Ansible scripts, accessible here, to automate the setup process. Ansible, an open-source suite, facilitates infrastructure-as-code through software provisioning, configuration management, and application deployment.

Subsequently, I set up the ELK stack and crafted a dashboard. The ELK stack, an acronym for Elasticsearch, Logstash, and Kibana, played a crucial role in automatically gathering logs on our ELK server and creating a comprehensive dashboard for monitoring.
