Android App

This project was part of my Web, Mobile and Security course, where Mr. Koreman provided valuable insights into mobile app vulnerabilities and tools for reverse engineering. Our task was to develop an Android app using the Kotlin programming language, a cross-platform, statically typed language with general-purpose high-level capabilities and type inference, closely resembling Java. More information about Kotlin can be found here.

For the project, we were required to create an Android app in Kotlin, incorporating at least one API call to a service of our choice and utilizing a database. After completing the app, we had to introduce a form of malware and create a Frida script capable of bypassing a specific function within our application.

In my app, the user logs in, and upon successful authentication, they can click the "Get random Pokémon" button. After a brief delay, an API call is made to the Pokemontcg API, displaying a random Pokémon card on the screen. I opted for SQLite3 as the database engine since it is pre-installed on every Android device. Information about the Pokemontcg API can be found here. Information about SQLite3 can be found here

To introduce malware, I implemented a feature: when the user clicks the Pokémon card button, all device information is sent to my Google Firebase database. To accomplish this, I rooted my phone to obtain the necessary permissions.

The Frida script I developed bypasses the login screen by setting the authentication function to true. This enables login without requiring a username or password. You can find the script here. For real-world implementation, the APK should be opened with the apktool, and after integrating the Frida server, it should be rebuilt. More information about that can be found here
